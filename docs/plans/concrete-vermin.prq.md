## Overview

Deliver Concrete Vermin: Tactical Reforged — the production game rebuild of `poc.html` — as a fully playable Android-first mobile game with 12 missions, 5 weapons, ~30-50 vermin variants, ~50 collectible artifacts, complete UI shell, audio pass, balance system, and shipping CI/CD.

## Phases (high-level)

| Phase | Goal | Tasks |
|---|---|---|
| 1 — Foundation | Repo runs `pnpm install && pnpm dev` and CI green on empty repo | CV-001 → CV-019 |
| 2 — Sim Core | Pure-TS sim with archetypes, traits, factories, scoring, encounter FSM, RNG | CV-020 → CV-039 |
| 3 — ECS Bridge | Koota world, traits, actions, systems wired to sim | CV-040 → CV-054 |
| 4 — Render | Pixi app, camera, sprite composition, effects (trails, splash, CRT, magnetism) | CV-055 → CV-074 |
| 5 — UI Shell | React shell, screens, HUD, cutscene reader, theme | CV-075 → CV-099 |
| 6 — Audio | Tone patches per weapon, vermin, ambience, music beds | CV-100 → CV-114 |
| 7 — Content | 12 missions, 30-50 variants, 50 collectibles, bestiary | CV-115 → CV-159 |
| 8 — Balance & Polish | Analysis system, governors, sweeps, autobalance, release gate, mobile QA | CV-160 → CV-189 |

## Dependency graph (high-level)

```
Phase 1 (foundation)
  → Phase 2 (sim core)
    → Phase 3 (ECS bridge)
      → Phase 4 (render) ‖ Phase 5 (UI shell) ‖ Phase 6 (audio)  [parallel]
        → Phase 7 (content) ‖ start of Phase 8 (analysis system + governors)
          → Phase 8 (balance lock pass + mobile QA + release wiring)
```

Within each phase, tasks parallelize where they don't share files. Specifically:

- **Phase 2**: CV-020/021/022 sequential; CV-023…CV-038 mostly parallel after CV-022
- **Phase 3**: CV-040 first; then CV-041…CV-053 mostly parallel
- **Phase 4**: CV-055/056/057 first; then effects + dioramas parallel
- **Phase 5**: CV-075/076/077/097 first; then screens + HUD parallel
- **Phase 7**: heavily parallel — every mission, variant, cutscene, and collectible is independent
